# Administrasjon i Active Directory

I dette steget jobber vi med:
- **OU (Organizational Units)** – struktur
- **Security Groups** – tilgang
- **GPO (Group Policy)** – regler og innstillinger


### Kort forklart

- **OU:** Mapper i AD som brukes til å organisere brukere og maskiner, og til å knytte GPO-er.
- **Security Group:** Samling av brukere som får samme tilgang (f.eks. til filer eller innlogging).
- **GPO:** Regler som gjelder for alle objekter i en OU (f.eks. skjermlås, bakgrunnsbilde).


## 1. Opprett OU-er

1. Gå til: Server Manager > Tools > Active Directory Users and Computers
2. Høyreklikk på domenet ditt → **New > Organizational Unit**
3. Opprett:
- `Brukere`
- `Grupper`

> Valgfritt: Lag under-OUer senere, f.eks. `Ansatte` og `Ledere`

---

## 2. Opprett en bruker

1. Høyreklikk på `Brukere` → **New > User**
2. Fyll inn navn og brukernavn  
Eksempel på vanlig måte å gjøre det: Simen Stenersen → simste

3. Sett passord:
- Minimum 7 tegn
- Må inneholde store/små bokstaver, tall og spesialtegn


## 3. Tillat innlogging på klienter (GPO)

For at brukere skal kunne logge inn på domenemaskiner, må vi lage en GPO.

### Lag GPO:

1. Gå til: Server Manager > Tools > Group Policy Management
2. Finn OU-en for klientmaskiner (eller domenet)
3. Høyreklikk → **Create a GPO in this domain and link it here**
4. Navn: f.eks. `Tillat innlogging`


### Konfigurer GPO:

1. Høyreklikk GPO → **Edit**
2. Gå til: Computer Configuration >
Policies >
Windows Settings >
Security Settings >
Local Policies >
User Rights Assignment >
Allow log on locally

3. Dobbeltklikk → **Add User or Group**
4. Legg til: Domain Users
5. Klikk **Apply** → **OK**

> 💡 Vi bruker **Domain Users** fordi det automatisk inkluderer alle brukere. OU-er som Domain Users kan ikke brukes direkte til sikkerhetsrettigheter.

Brukere du har opprettet i AD kan nå logge inn på klientmaskiner i domenet!
