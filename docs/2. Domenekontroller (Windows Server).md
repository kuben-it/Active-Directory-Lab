# Introduksjon
**Domenekontrolleren** er serveren som styrer klientene og tjenestene i nettverket ved hjelp av Active Directory. For eksempel ligger alle klientenes brukerkontoer og deres tilganger lagret hos domenekontrolleren.
I vårt oppsett skal den også tilby DNS og DHCP.
Vi setter den opp som en virtuell maskin med Windows Server, så du trenger en ISO-fil for dette operativsystemet.

Veldig god forklaring på hva en domenekontroller er og hvorfor det brukes hos [NDLA](https://ndla.no/nb/r/driftsstotte-im-itk-vg2/datalab-med-windows-server-og-generisk-nettverk-utvidet/c3aa06b3d4/15128).

# Instruksjoner

## 1. Lag ny VM i Hyper-V
Åpne Hyper-V Manager og velg New > Virtual Machine

### Anbefalte innstillinger:
* Navn: DC1
* Generation: Generation 2
* Memory: 2048 MB
* Networking: svitsjen du lagde i forrige steg
* Harddisk: bruk default innstillinger, men sett diskstørrelsen til 64 GB
* Installation options: Installer fra .iso > Windows Server ISOen du lastet ned tidligere
* Fullfør

## 2. Installer Windows Server
* Start VM-en
* Velg språk og tastatur (⚠️lurt å velge norsk keyboard)
* Velg versjon: Windows Server 2022 Standard (Desktop Experience). ⚠️ **OBS: ganske viktig at du velger riktig versjon!**
* Velg "Custom installation" > Drive 0
* Når installasjonen er ferdig angir du et administrator-passord, ✏️ **som det er viktig at du skriver ned og lagrer på et lurt sted** ✏️
* Før vi går videre er det lurt å bytte navnet på maskinen til "DC1" og restarte maskinen:

## 3. Gi serveren statisk IP
* Åpne "Ethernet settings" > Change Adapter Options > Høyreklikk > Properties > IPv4 > Properties

![ethernetsettings](https://github.com/user-attachments/assets/7d271e14-1348-4372-bda9-f17d890fd28a)![ip_innstillinger](https://github.com/user-attachments/assets/4f85e2ae-1b12-4350-a7ad-33ea57def48e)

* IP: 192.168.10.1
* Subnet mask: 255.255.255.0
* Default gateway: 192.168.10.1
* Preferred DNS server: 127.0.0.1

## 4. Installer tjenester: Active Directory + DNS + DHCP
* Åpne Server Manager > Add Roles and Features > Next til du kommer til **Server Roles**
* Her velger du "Active Directory Domain Services", "DHCP Server" og "DNS Server" i tillegg til det som er valgt som standard, og fullfører veiviseren med "install".
* Restart maskinen når installasjonen er ferdig.

## 5. Gjør serveren til Domain Controller
⚠️ ***Her er det fort gjort å gjøre ganske alvorlige feil, så lag et nytt "checkpoint" i Hyper-V før du går videre. Det er dessverre også viktig at du gjør ting i riktig rekkefølge.***  ⚠️

Her kommer det noen nye begreper. En "forest" er det øverste nivået i en AD-struktur. Vi bruker .local som et slags "falskt" top-level domene til testing. **Instruksjoner**:

* I Server Manager vil du nå se et varsel oppe til høyre. Klikk der og så på "Promote this server to a domain controller"
* Add a new forest > kall den "[velgetnavn].[toppdomene]", f.eks. minbedrift.local. (Her kan det komme en rar feilmelding du kan ignorere)
    * Domenenavnet kan være hva du vil, men toppdomentet må være en av disse: .local, .intranet, .internal, .private, .corp, .home, .lan. Disse er forbeholdt testing og lokal bruk. 
* La alt annet stå, men skriv inn et DSRM-passord som du noterer ned der du lagrer teknisk dokumentasjon. Passordet brukes til å gjenopprette domenet hvis noe går galt.
* Klikk videre (ignorer varsel på "DNS options", vi fikser DNS senere), installer alt som må installeres, du trenger ikke endre på noe. Til slutt vil maskinen restarte av seg selv.
* Hvis det har gått riktig, burde du se domenenavnet når du logger inn!

PS: grunnen til at vi setter opp DNS lokalt:
1. Kunne navngi enheter på nettverket så vi ikke trenger å huske IP-adresser. 
2. Fordi Active Directory krever DNS for å fungere. Det er gjennom DNS at klientene finner domenekontrolleren og andre tjenester i nettverket.
